Почему именно Angular

  • Двухстороннее связывание данных
  • Поведение описывается директивами
  • Гибкая система фильтров

Директивы

Vanilla.js

			<button id="save">Сохранить</button>
			<script>
					if (document.exist) $("#save").show();
			</script>

Angular.js

				<button ng-show="document.exist">Сохранить</button>

Двухстороннее связывание данных

{{d.title}} : {{d.content}}

Как работает двухсторонее связывание данных

			DocumentFormController = ($scope) ->
				$scope.save = () -> 
					$scope.documents.push $scope.document
					$scope.document = {title:"",content:""}
				$scope.documents = []

Как работает двухсторонее связывание данных

			<div ng-controller="DocumentFormController">
				<input ng-model="document.title"></input>
				<textarea ng-model="document.content"></textarea>
				<button ng-click="save()">Сохранить</button>
				<article ng-repeat="d in documents">
					<hr>{{d.title}} : {{d.content}}
				</article>
			</div>

Сервисы

  • Service - самый простой тип, по сути является классом
  • Factory - конструирует значение и передает его
  • Provider - доступен для настройки еще до запуска приложения

Service

Сервис это просто класс

			class HelloService
				hello: -> "Hello, world"
			app.service("hello", HelloService)
			HelloController = (hello) ->
				# hello == new HelloService

Factory

Фабрика конструирет значение и возвращает его

			app.factory "hello", ->
				{hello: "Hello world"}
			HelloController = (hello) ->
				# hello == {hello: "Hello world"}

Provider

			app.provider "hello", class
				$get: -> "Hello, #{@name}"
				name: "World"
				setName: (@name) ->
			app.config (helloProvider) ->
				helloProvider.setName "Artem"
			HelloController = (hello) ->
				# hello == "Hello, Artem"
